## 五、代码 Bug 和潜在问题

### 严重 (P0)

| # | 文件:行 | 问题 | 状态 |
|---|---------|------|------|
| B1 | `src/dao/d1.ts:214-221` | **`saveMessage` 无事务**：INSERT message + UPDATE threads 是两条独立 SQL，D1 故障时可能只执行第一条。应用 `db.batch()` 包装 | ⏳ 他人修复 |
| B2 | `src/dao/d1.ts:468-479` | **FTS 搜索不过滤 user_id**：`searchFTS` 查询没有 `WHERE user_id = ?`，所有用户的文档都会被搜索到。Qdrant 向量搜索有 user_id 过滤，但 FTS 层可能泄露跨用户数据 | 待修复 |
| B3 | `src/services/parser.ts:42` | **大图片 OCR 可能崩溃**：`btoa(String.fromCharCode(...new Uint8Array(buffer)))` 对大文件（接近 10MB）可能在 Worker 中因内存超限崩溃，`String.fromCharCode` 对超长数组有栈溢出风险 | 待修复 |

### 中等 (P1)

| # | 文件:行 | 问题 |
|---|---------|------|
| B4 | `src/agent/tools.ts:278` + `src/dao/d1.ts:424` | **`getExpiresAt` 重复定义**：tools.ts 和 d1.ts 各有一份完全相同的函数，违反 DRY |
| B5 | `src/services/upload.ts:46-63` | **`saved` 可能为 null**：`insertChunk` 返回 `Chunk|null`，但后续直接访问 `saved.id` 未做 null 检查 |
| B6 | `src/index.ts:88-91` | **`/api/user/update` 无 ownership 检查**：任何已认证用户可修改任意用户的 name/ai_nickname |
| B7 | `src/config.ts:46-56` | **`allowedMimeTypes` 死代码**：upload 路由只校验 `allowedExtensions`，不校验 MIME type |

### 低 (P2)

| # | 文件:行 | 问题 |
|---|---------|------|
| B8 | `src/agent/tools.ts:296` | **memory_save 去重阈值 0.95**：cosine similarity 0.95 可能过于严格，大量相似但不完全重复的记忆被当作重复处理 |
| B9 | `public/app.js:375` | **`updateDisplayedNames` 空 userName 崩溃**：`userName[0].toUpperCase()` — 如果 userName 为空字符串会崩溃（M10 修复了 avatar 处，此处未同步修复） |
