1. 项目背景

请你基于 Node.js 或 Python 或其他（不建议使用 Java），开发一个智能对话式任务管理助手，并部署到 Cloudflare Worker 上，提供一个可公开访问的网址。

本项目旨在考察你对 AI Agent 核心概念（工具调用、记忆召回、子代理规划）的掌握程度，以及实际工程落地能力。要求代码模块拆分清晰、简洁优雅，体现技术审美与品味。


---

2. 核心功能需求（必须完成）

2.1 基础对话与用户管理
- 实现一个网页对话界面（可使用现成框架，如 Node.js 的 ChatUI、Vercel AI SDK，或 Python 的 Chainlit、Gradio、Streamlit）。
- 当用户未提供邮箱和姓名时，AI 主动询问并存储。
- 后续对话中 AI 能正确称呼用户。
- 用户可以为 AI 设置昵称，并支持修改。
  
2.2 任务管理（vibe coding 方式）
- 用户通过自然语言与 AI 对话，增删改查自己的任务列表以及列表中任务的具体需求。
- 不依赖传统表单，完全通过对话完成操作。
  
2.3 外部搜索与深度研究
- 接入 Serper.dev 或其他搜索 API，支持实时信息检索。
- 当用户提出复杂研究主题时，AI 能进行多轮搜索、规划子代理（将任务拆解为多个子任务），并整合生成结构化报告。
  
2.4 文件处理与 RAG 系统
- 用户可上传文件（如 PDF、Word、TXT、Markdown 等）。
- 系统需对文件进行自动解析、分块、向量化，并存储到向量数据库（如 Qdrant 免费云）。
- 后续对话中，AI 能通过语义检索（RAG） 召回文件中的相关内容，并作为上下文增强回答。
- 支持对上传文件的增删改查（用户可在工作空间中管理文件）。
  
2.5 数据持久化
- 使用 Cloudflare D1（或类似关系型数据库）存储用户信息、任务列表、对话记录等。
- 使用向量数据库（如 Qdrant）存储文档片段与对话记忆。
  

---

3. 加分项（可选，可显著提升评价）

- 多模态大模型应用：支持用户上传图片、音频或视频文件，并能从中提取信息（如 OCR 识别图片文字、音频转文字、视频关键帧分析等），将提取结果纳入 RAG 检索或对话上下文。
- 实现 Tree of Thoughts (TOT) 或 Graph of Thoughts (GOT) 推理模式，并展示其效果。
- 提供完整的前端工作空间（文件浏览器、上传进度条、文件类型标签）。
- 实现细粒度的意图识别与动态 Prompt 模板选择。
- 完善的单元测试与部署流水线。
  

---

4. 技术要求与限制

- 运行环境：Cloudflare Worker（必须部署到 Cloudflare 边缘网络）。
- AI 模型：可使用 Gemini 2.0 Flash Lite（或其他免费/低成本模型），需实现 LLM 抽象层以便未来切换。
- 向量数据库：Qdrant Cloud 免费层等。
- 关系数据库：Cloudflare D1 等。
- 前端：不限框架，但需提供可访问的网页链接。
- 代码仓库：提供 GitHub/GitLab 仓库链接，README 中包含部署说明和访问地址。
