# Project Memory

## Key Decisions

- **User registration**: Original requirement §2.1 says AI should proactively ask for email/name. Changed to form-based registration for simpler UX and reliability. Recorded in spec §1.
- **Architecture**: CF Worker monolith with Hono framework. Single Worker handles API + static files. No separate Queue service.
- **LLM**: GLM API (OpenAI-compatible) as primary LLM + embedding provider.
- **Vector DB**: Qdrant in hybrid mode (local dev + cloud production). Dual-write with outbox safety net for D1↔Qdrant consistency. Collection name configurable via `QDRANT_COLLECTION` env var (local: `xvc_agent_chunks`).
- **Frontend**: Pure HTML + Vanilla JS, no build step.
- **Deep research**: Currently uses prompt-guided multi-round tool calls. AgentLoop refactored to AsyncGenerator pattern (Phase 1 done). Planned to switch to spawn_agent after Phase 2. See `docs/superpowers/specs/2025-04-17-spawn-agent-design.md`.
- **时区**: 全系统统一使用东八区（UTC+8）时间，包括数据库存储和页面显示。`datetime('now')` 需改为 `datetime('now', '+8 hours')`。

### Source Code
- `src/index.ts` — Hono app with all API routes + static file serving
- `src/agent/loop.ts` — AgentLoop: AsyncGenerator-based (`execute()` yields `AgentEvent`). `run()` is SSE thin wrapper with flush logic. `agentId` on constructor.
- `src/agent/tools.ts` — 9 tool definitions + convention-based dispatch (spawn_agent planned for Phase 2)
- `src/agent/prompt.ts` — System prompt builder (tools→instructions→user→RAG→datetime)
- `src/dao/d1.ts` — All D1 operations (users, tasks, threads, messages, documents, chunks, FTS)
- `src/dao/qdrant.ts` — Qdrant HTTP API (ensureCollection, upsert, search, delete)
- `src/dao/outbox.ts` — Outbox event management for D1↔Qdrant consistency
- `src/llm/client.ts` — LLMClient with SSE streaming + tool_calls parsing
- `src/llm/embedding.ts` — EmbeddingClient (GLM embedding API)
- `src/services/cjk.ts` — CJK tokenizer via Intl.Segmenter
- `src/services/cleaner.ts` — Text cleaning (whitespace, control chars, HTML, NFC)
- `src/services/chunker.ts` — Heading-aware chunker (~500 tokens, 15% overlap)
- `src/services/parser.ts` — File parser (PDF, DOCX, TXT, MD → text) with dynamic imports
- `src/services/search.ts` — Hybrid search: FTS5 + Qdrant → RRF fusion
- `src/services/upload.ts` — File upload pipeline (R2 → parse → clean → chunk → embed → Qdrant)
- `src/services/web.ts` — Serper search + URL fetch
- `src/middleware/auth.ts` — X-User-Id validation middleware
