## Key Decisions

- **User registration**: Original requirement ¬ß2.1 says AI should proactively ask for email/name. Changed to form-based registration for simpler UX and reliability. Recorded in spec ¬ß1.
- **Architecture**: CF Worker monolith with Hono framework. Single Worker handles API + static files. No separate Queue service.
- **LLM**: GLM API (OpenAI-compatible) as primary LLM provider. SiliconFlow (BAAI/bge-m3) for embedding.
- **Vector DB**: Qdrant in hybrid mode (local dev + cloud production). Dual-write with outbox safety net for D1‚ÜîQdrant consistency. Collection name configurable via `QDRANT_COLLECTION` env var (local: `xvc_agent_chunks`).
- **Frontend**: Pure HTML + Vanilla JS, no build step.
- **Deep research**: spawn_agent tool implemented. Sub-agents execute in parallel, results synthesized into structured report via prompt guidance. See `docs/superpowers/specs/2025-04-17-spawn-agent-design.md`.
