# Project Memory

## Key Decisions

- **User registration**: Original requirement ¬І2.1 says AI should proactively ask for email/name. Changed to form-based registration for simpler UX and reliability. Recorded in spec ¬І1.
- **Architecture**: CF Worker monolith with Hono framework. Single Worker handles API + static files. No separate Queue service.
- **LLM**: GLM API (OpenAI-compatible) as primary LLM provider. SiliconFlow (BAAI/bge-m3) for embedding.
- **Vector DB**: Qdrant in hybrid mode (local dev + cloud production). Dual-write with outbox safety net for D1вЖФQdrant consistency. Collection name configurable via `QDRANT_COLLECTION` env var (local: `xvc_agent_chunks`).
- **Frontend**: Pure HTML + Vanilla JS, no build step.
- **Deep research**: Currently uses prompt-guided multi-round tool calls. AgentLoop refactored to AsyncGenerator pattern (Phase 1 done). Planned to switch to spawn_agent after Phase 2. See `docs/superpowers/specs/2025-04-17-spawn-agent-design.md`.
